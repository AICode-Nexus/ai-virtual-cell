Contribute

最适合参与的角色

生物学家

判断机制迁移是否抓住了真正有价值的控制逻辑，而不是只做术语类比。

AI 研究者

评估这些控制器与现有方法相比，是否有可证明的性能或能力差异。

系统架构师

帮助把机制原型变成可组合的系统层，而不是零散实验脚本。

安全研究者 / 产品研究者

分别验证对齐机制与应用场景，帮助判断哪些方向最有现实价值。

如何进入讨论

1

先选文档入口

先从设计、映射或原型三个入口里选一个，不要一开始就横向扫所有文件。

2

带着具体问题来

例如“这条映射是否过度简化”“这个控制器是否能替代现有方案”。

3

在 Discussions 或 Issues 留下结论

把反馈落成具体判断，而不是停留在“这个很有意思”。

参与 Discussions 提交 Issues

本地开始方式

# clone repo
git clone https://github.com/AICode-Nexus/ai-virtual-cell.git
cd ai-virtual-cell

# read the core docs
cat docs/DESIGN.md
cat docs/BIOLOGY_MAPPING.md
cat docs/PROJECT_COMPLETION_REPORT.md

# run innovation demos
python3 src/innovations/demo_integrated.py

# preview site locally
cd website && python3 -m http.server 8000
Topics

建议优先讨论的话题

生物学验证

哪些映射是强控制逻辑迁移，哪些只是弱类比。

技术实现

哪些机制最适合先做成算法，哪些更像系统控制层。

应用场景

哪些真实系统最需要持续学习、动态对齐和分层容错。

新映射方向

继续扩展表观遗传、代谢网络、细胞通讯等方向。
